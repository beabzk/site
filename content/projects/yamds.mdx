# YAMDS - Yet Another Markdown Documentation System

A modern documentation generator with live preview and theme support, built for developers who want beautiful docs without the complexity.

<Callout type="info" title="Why YAMDS?">
  While there are many documentation generators available, YAMDS focuses on
  simplicity and developer experience. It's designed to get out of your way and
  let you focus on writing great documentation.
</Callout>

## Installation

```bash
npm install -g yamds
```

## Quick Start

```bash
# Initialize a new documentation site
yamds init my-docs
cd my-docs

# Start the development server
yamds dev

# Build for production
yamds build
```

<Callout type="success" title="Quick Setup">
  YAMDS can have you up and running with beautiful documentation in under 2
  minutes!
</Callout>

## Configuration

YAMDS uses a simple configuration file:

```typescript
// yamds.config.ts
export default {
  title: "My Documentation",
  description: "Comprehensive documentation for my project",
  theme: "default",
  sidebar: [
    {
      text: "Getting Started",
      items: [
        { text: "Introduction", link: "/intro" },
        { text: "Installation", link: "/installation" },
      ],
    },
  ],
  plugins: ["search", "analytics"],
};
```

## Themes

YAMDS comes with several built-in themes:

### Default Theme

Clean and modern design with excellent readability.

### Minimal Theme

Stripped down for maximum focus on content.

### Dark Theme

Dark mode optimized for late-night documentation sessions.

### Technical Theme

Code-focused design perfect for API documentation.

```bash
# Switch themes easily
yamds build --theme minimal
```

## Plugin System

Extend YAMDS with powerful plugins:

```typescript
// yamds.config.ts
export default {
  plugins: [
    "search", // Full-text search
    "analytics", // Google Analytics
    "sitemap", // XML sitemap generation
    "social-cards", // Auto-generate social media cards
    "code-copy", // Copy code button
    "mermaid", // Diagram support
  ],
};
```

### Custom Plugins

```typescript
// plugins/custom-plugin.ts
export default function customPlugin(options: any) {
  return {
    name: "custom-plugin",
    setup(build: any) {
      // Plugin logic here
    },
  };
}
```

## Deployment

Deploy to any static hosting provider:

```bash
# Build for production
yamds build

# Deploy to Vercel
vercel deploy dist

# Deploy to Netlify
netlify deploy --prod --dir dist

# Deploy to GitHub Pages
yamds deploy --provider github-pages
```
